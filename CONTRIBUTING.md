# Contributing to Portal

Thanks for your interest in Portal. This guide covers the repo layout, local development, and the
checks a change needs to pass.

## Repo layout

```
apps/api/    FastAPI backend + the arq sync worker (Python, managed with uv)
  src/portal/  application package        alembic/  database migrations
apps/web/    SvelteKit 5 frontend (admin UI + public /u and /d pages), Tailwind v4
docker/      dev compose + the all-in-one image's build assets (supervisord, entrypoint, …)
docs/        deployment + Frame.io integration docs
Dockerfile          the all-in-one farbhaus/portal image (build context = repo root)
docker-compose.yml  production single-service deploy; .env.example alongside it
```

## Local development

`make dev` is the production deploy with a `--build` overlay — it builds the all-in-one image
from source and runs it from the same `docker-compose.yml` + `.env` (so create `.env` first, e.g.
`cp .env.example .env`):

```bash
make dev      # docker compose -f docker-compose.yml -f docker/docker-compose.dev.yml up -d --build
```

For day-to-day work on a single service you can also run the backend and frontend directly
(below), which is faster than rebuilding the image. Run `make` to see all targets.

### Backend (apps/api)

Requires Python 3.12+ and [uv](https://docs.astral.sh/uv/).

```bash
cd apps/api
uv sync                       # create the venv from uv.lock

uv run ruff check             # lint
uv run mypy src               # types (strict)
uv run pytest                 # tests
```

The test suite needs a throwaway Postgres on **:55432** and Redis on **:56379** (hardcoded in
`tests/conftest.py`, which drops all tables — never point it at a real database):

```bash
docker run -d --rm -p 55432:5432 -e POSTGRES_USER=portal -e POSTGRES_PASSWORD=portal \
  -e POSTGRES_DB=portal --name portal_test_pg postgres:16
docker run -d --rm -p 56379:6379 --name portal_test_redis redis:7
```

New/changed Alembic migrations are autogenerated against a live Postgres — see `apps/api/alembic`.

### Frontend (apps/web)

Requires Node 22.

```bash
cd apps/web
npm install
npm run check                 # svelte-check (must be 0 errors)
npm run build                 # production build
npm run dev                   # local dev server
```

`npm run gen:api` regenerates the typed API client (`src/lib/api/schema.d.ts`) from the running
API's OpenAPI schema; run it after adding/changing backend routes.

## Before you open a PR

All of these must be green:

```bash
make check    # ruff + mypy (apps/api) + svelte-check (apps/web)
make test     # backend test suite (spins throwaway Postgres/Redis)
```

- Branch from and target **`dev`** (releases promote `dev` → `main`).
- Match the surrounding code's style and comment density; keep changes focused.
- Update docs when you change deploy/config behavior.
